go-tour

发表于 2021-09-19 分类于 笔记 本文字数： 1.7k 阅读时长 ≈ 2 分钟

这是对go-tour指南的一篇小笔记

  1. 在 Go 中，如果一个名字以大写字母开头，那么它就是已导出的。在导入一个包时，你只能引用其中已导出的名字
  2. 函数外的每个语句都必须以关键字开始（var, func 等等），因此 := 结构不能在函数外使用。
  3. 一个未指定类型的常量由上下文来决定其类型。
  4. defer 语句会将函数推迟到外层函数返回之后执行。推迟的函数调用会被压入一个栈中
  5. 切片并不存储任何数据,更改切片的元素会修改其底层数组中对应的元素。
  6. b := make([]T, len, cap T)
  7. func append(s []T, vs …T) []T
  8. func copy(dst, src []T) int
    copy 函数支持不同长度的切片之间的复制（它只复制较短切片的长度个元素）。 此外， copy 函数可以正确处理源和目的切片有重叠的情况。
  9. make 函数会返回给定类型的映射，并将其初始化备用。(必须使用)当从映射中读取某个不存在的键时，结果是映射的元素类型的零值。
  10. Go 函数可以是一个闭包。闭包是一个函数值，它引用了其函数体之外的变量。该函数可以访问并赋予其引用的变量的值，换句话说，该函数被这些变量“绑定”在一起。
  11. 方法就是一类带特殊的 接收者 参数的函数。方法接收者在它自己的参数列表内，位于 func 关键字和方法名之间。(方法只是个带接收者参数的函数。)
    注：接收者的类型定义和方法声明必须在同一包内；不能为内建类型声明方法。
  12. 指针接收者的方法可以修改接收者指向的值；若使用值接收者，那么方法会对值的副本进行操作。
  13. 以指针为接收者的方法被调用时，接收者既能为值又能为指针(这不同于函数参数必须对应)。接受者为值时，go编译器会修改其为指针接受者。同样的，在以值为接收者的方法被调用时，接收者既能为值又能为指针。
  14. 接口类型的变量可以保存任何实现了这些方法的值。
  15. 类型通过实现一个接口的所有方法来实现该接口
  16. 在内部，接口值可以看做包含值和具体类型的元组：(value, type)
    接口值保存了一个具体底层类型的具体值。
    接口值调用方法时会执行其底层类型的同名方法。
  17. 为 nil 接口调用方法会产生运行时错误，因为接口的元组内并未包含能够指明该调用哪个具体方法的类型。
  18. 指定了零个方法的接口值被称为 空接口： interface{}
    空接口可保存任何类型的值。(因为每个类型都至少实现了零个方法)
    空接口被用来处理未知类型的值。
  19. 类型断言：
    • t := i.(T)
      该语句断言接口值 i 保存了具体类型 T，并将其底层类型为 T 的值赋予变量 t。
      若 i 并未保存 T 类型的值，该语句就会触发一个恐慌。
    • t, ok := i.(T)
      若 i 保存了一个 T，那么 t 将会是其底层值，而 ok 为 true。否则，ok 将为 false 而 t 将为 T 类型的零值，程序并不会产生恐慌。
  20. 类型选择：
    switch v := i.(type)
  21. fmt 包中定义的 Stringer 是一个可以用字符串描述自己的类型。fmt 包（还有很多包）都通过此接口来打印值。
    可通过重写此接口改变fmt.Println的输出
  22. io.Reader 接口有一个 Read 方法：
    func (T) Read(b []byte) (n int, err error)
    包含在io包中
  23. go f(x, y, z)会启动一个新的 Go 程并执行f(x, y, z)
    f, x, y 和 z 的求值发生在当前的 Go 程中，而 f 的执行发生在新的 Go 程中
    Go 程在相同的地址空间中运行，因此在访问共享的内存时必须进行同步。
  24. 信道是带有类型的管道，你可以通过它用信道操作符 <- 来发送或者接收值。
    ch := make(chan int) 信道在使用前必须被创建、
    默认情况下，发送和接收操作在另一端准备好之前都会阻塞
  25. 信道可以是 带缓冲的.仅当信道的缓冲区填满后，向其发送数据时才会阻塞。当缓冲区为空时，接受方会阻塞
  26. 接收者可以通过为接收表达式分配第二个参数来测试信道是否被关闭：若没有值可以接收且信道已被关闭，
    那么在执行完v, ok := <-ch
    之后 ok 会被设置为 false
    循环 for i := range c 会不断从信道接收值，直到它被关闭
    注意： 只有发送者才能关闭信道，而接收者不能。向一个已经关闭的信道发送数据会引发程序恐慌（panic）。
